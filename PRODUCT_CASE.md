# AI-ассистент для пользователей с СДВГ

## Описание проекта

Этот проект представляет собой продуктовый кейс разработки AI-ассистента, который помогает пользователям с СДВГ сохранять контекст задач и продолжать работу при снижении энергии и концентрации.

Основная идея продукта — адаптировать рекомендации ассистента в зависимости от состояния пользователя, чтобы снизить когнитивную нагрузку и помочь пользователю продолжать выполнение задачи.

Ассистент учитывает:

- уровень энергии пользователя
- эмоциональное состояние
- доступное время
- текущий этап выполнения задачи

На основе этих параметров система предлагает небольшие управляемые шаги, которые позволяют пользователю продолжить выполнение задачи без перегрузки.

---

# Проблема

Пользователи с СДВГ часто сталкиваются со следующими трудностями:

- потеря контекста текущей задачи
- снижение энергии и мотивации
- когнитивная перегрузка
- сложность возвращения к задаче после перерыва

Большинство AI-ассистентов дают универсальные ответы и не учитывают текущее состояние пользователя.

Это приводит к тому, что рекомендации оказываются слишком сложными или нерелевантными.

---

# Решение

AI-ассистент, который адаптирует ответы в зависимости от состояния пользователя.

Ассистент учитывает:

- уровень энергии
- эмоциональное состояние
- доступное время
- текущий этап выполнения задачи

Главная идея продукта — помогать пользователю продолжить работу через небольшие управляемые действия.

# Product Rules

Набор правил, определяющих поведение ассистента.

Основные правила:

- избегать сложных рекомендаций при низкой энергии пользователя
- предлагать небольшие управляемые шаги
- сохранять контекст текущей задачи
- не перегружать пользователя большим количеством действий
- помогать пользователю продолжать выполнение задачи

---

# Emotional Layer

Слой адаптации ответа в зависимости от состояния пользователя.

Логика работы:

низкая энергия → простые действия  
фрустрация → поддержка и упрощение задачи  
высокая энергия → следующий шаг задачи  

Цель слоя — снизить когнитивную нагрузку пользователя и повысить вероятность продолжения задачи.

# Пользовательские сценарии

## Сценарий 1 — пользователь потерял фокус

Пользователь:

"Я не могу продолжить писать отчёт."

Ассистент:

1 анализирует состояние пользователя  
2 определяет низкий уровень энергии  
3 предлагает небольшой следующий шаг  

Ответ:

"Похоже сейчас сложно продолжать. Попробуем маленький шаг: напишите только заголовок следующего раздела."

---

## Сценарий 2 — перегрузка задачами

Пользователь:

"У меня слишком много задач."

Ассистент:

- помогает разбить задачу
- предлагает приоритет
- снижает когнитивную нагрузку

---

## Сценарий 3 — возвращение к задаче

Пользователь возвращается к задаче после перерыва.

Ассистент:

- восстанавливает контекст
- предлагает следующий шаг
- снижает барьер входа в задачу

# North Star Metric и система KPI продукта

## 1. North Star Metric

**North Star Metric (NSM)** — это ключевая метрика, которая показывает, что продукт действительно создаёт ценность для пользователя.

Для продукта — AI-ассистента, который помогает пользователю **начать и завершить действие**, North Star Metric можно сформулировать следующим образом:

**Weekly Completed Actions (WCA)**  
Количество действий, которые пользователи начали и довели до завершения с помощью продукта за неделю.

Эта метрика отражает реальную ценность продукта:
- пользователь начал действие
- пользователь не потерял контекст
- пользователь довёл действие до результата

---

## 2. Формула North Star

NSM можно разложить на три компонента:

NSM = Users × Actions Started × Completion Rate

где:

- **Users** — количество активных пользователей  
- **Actions Started** — количество начатых действий  
- **Completion Rate** — доля завершённых действий  

---

## 3. KPI система вокруг North Star

### 1. Acquisition (Привлечение)

Метрики роста аудитории:

- New Users per Week
- Cost per Acquisition (CPA)
- Install → Registration Conversion

---

### 2. Activation (Активация)

Показывает, понял ли пользователь ценность продукта.

Ключевое событие — **первое начатое действие**.

KPI:

- Activation Rate  
  (пользователи, начавшие первое действие / новые пользователи)

- Time to First Action

---

### 3. Engagement (Использование продукта)

Показывает регулярность использования.

KPI:

- Daily Active Users (DAU)
- Weekly Active Users (WAU)
- Actions Started per User

---

### 4. Core Value (Ценность продукта)

Метрики, напрямую связанные с North Star.

KPI:

- Action Completion Rate
- Completed Actions per User
- Average Action Duration
- Context Loss Rate

---

### 5. Retention (Удержание пользователей)

Показывает, возвращаются ли пользователи.

KPI:

- Day 1 Retention
- Day 7 Retention
- Week 4 Retention

---

## 4. Итог

**North Star Metric продукта:**

Weekly Completed Actions — количество действий, завершённых пользователями с помощью продукта за неделю.

Эта метрика отражает ключевую ценность продукта — способность помогать пользователям **начинать и доводить до конца действия, не теряя контекст**.
